AFTER inputs: services: type: string ecr_registry: # 🆕 ADDED type: string default: '' aws_region: # 🆕 ADDED type: string default: us-east-1 build_args: # 🆕 ADDED type: string default: ''

Before: 3 stub jobs that did nothing (exit 1). No deployment logic existed.

Job Structure

BEFORE jobs: deploy-prod: steps: - run: exit 1 # ❌ Stub 1 - run: exit 1 # ❌ Stub 2 - run: exit 1 # ❌ Stub 3
AFTER (12 Steps) jobs: deploy-prod: steps: 1. OIDC auth ✓ 2. Verify backend image ✓ 3. Verify frontend image ✓ 4. Install kubectl ✓ 5. Get kubeconfig ✓ 6. Smoke test access ✓ 7. Create K8s Secret ✓ 8. kustomize edit image ✓ 9. kubectl apply ✓ 10. Wait rollout ✓ 11. Resolve ALB ✓ 12. Health check ✓

Key Steps Added

Step 1-3: OIDC Auth + Image Verification
🆕 NEW - name: Configure AWS credentials (OIDC) uses: aws-actions/configure-aws-credentials@v6 with: role-to-assume: ${{ secrets.AWS_ROLE_ARN }} - name: Verify backend image exists in ECR # 🆕 Fail-fast run: aws ecr describe-images --repository-name ums-backend ... - name: Verify frontend image exists in ECR # 🆕 Fail-fast run: aws ecr describe-images --repository-name ums-frontend ...
Step 4-6: kubectl Setup + Access Check
🆕 NEW - name: Set up kubectl (v1.32) run: curl -LO ... kubectl - name: Update kubeconfig run: aws eks update-kubeconfig ... - name: Verify namespace access # 🆕 Smoke test run: kubectl auth can-i get pods --namespace ums
Step 7: Create K8s Secret
🆕 NEW - name: Create/update app secrets run: kubectl create secret generic ums-secrets \ --from-literal=SENDGRID_API_KEY=... \ --from-literal=OTP_HASH_SECRET=... \ --from-literal=ADMIN_BEARER_TOKEN=... \ --dry-run=client -o yaml | kubectl apply -f -
Step 8-10: kustomize + Apply + Rollout
🆕 NEW - name: Set image tags in kustomization run: kustomize edit set image \ ums-backend=ecr.../ums-backend:sha-... \ ums-frontend=ecr.../ums-frontend:sha-... - name: Apply manifests run: kubectl apply -k k8s/overlays/prod - name: Wait for backend rollout (3 min timeout) run: kubectl rollout status deployment/ums-backend --timeout=3m - name: Wait for frontend rollout (2 min timeout) run: kubectl rollout status deployment/ums-frontend --timeout=2m
Step 11-12: ALB Resolution + Health Check
🆕 NEW - name: Resolve ALB endpoint # 🆕 Poll up to 5 min run: for i in {1..30}; do ALB_HOST=$(kubectl get ingress ums -o jsonpath='{.status.loadBalancer.ingress[0].hostname}') [ -n "$ALB_HOST" ] && exit 0 sleep 10 done - name: Health check (with retry) # 🆕 Retry 15x, 2.5 min run: for i in {1..15}; do curl -f http://$ALB_HOST/health && exit 0 sleep 10 done - name: Dump diagnostics on failure # 🆕 Debug info if: failure() run: kubectl cluster-info; kubectl get pods ...
